Каскад стилей

Каскадирование - это механизм, который управляет конечными значениями свойств элемента, если к нему применяется несколько CSS-правил.

  • Если к элементу применяется несколько правил, их свойства комбинируются
  • Если в правилах есть одинаковые свойства с разными значениями, то они конфликтуют
/* Каким будет цвет текста абзацев? */
p {
  color: blue;
  background-color: orange;
}

p {
  color: teal;
}

Для того, чтобы собрать финальные стили элемента и разрешить конфликты значений свойств, браузер использует два механизма: специфичность и наследование.

Специфичность селекторов​

Для каждого CSS-правила браузер вычисляет специфичность (вес) селектора. Если к элементу нужно применить свойства из разных правил, и есть конфликтующие (одинаковые) свойства, используется значение свойства из правила с наиболее высокой специфичностью селектора.

Значение специфичности состоит из четырёх рангов, важность которых увеличивается справа налево.

Селекторы 4-го ранга​

Каждый селектор элемента и псевдоэлемента добавляет единицу в четвёртый ранг.

<article>
  <h1>Lorem ipsum dolor sit amet</h1>
  <p>
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a href="">Читать дальше...</a>
</article>

Цвет текста абзаца будет оранжевым, потому что специфичность второго правила выше.

/* Специфичность - 0 0 0 1 */
p {
  color: green;
}

/* ✅ Специфичность - 0 0 0 2 */
article > p {
  color: orange;
}

Селекторы 3-го ранга​

Каждый селектор класса, атрибута и псевдокласса добавляет единицу в третий ранг.

<article class="post">
  <h1 class="post-title">Lorem ipsum dolor sit amet</h1>
  <p>
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a href="" class="post-link">Читать дальше...</a>
</article>

Цвет текста заголовка будет оранжевым, потому что специфичность третьего правила выше.

/* Специфичность - 0 0 1 0 */
.post-title {
  color: green;
}

/* Специфичность - 0 0 1 1 */
.post > h1 {
  color: red;
}

/* ✅ Специфичность - 0 0 2 0 */
.post > .post-title {
  color: orange;
}

Цвет текста ссылки будет коричневым, потому что специфичность пятого правила выше.

/* Специфичность - 0 0 0 1 */
a {
  color: green;
}

/* Специфичность - 0 0 1 0 */
.post-link {
  color: orange;
}

/* Специфичность - 0 0 1 1 */
a.post-link {
  color: blue;
}

/* Специфичность - 0 0 2 0 */
.post > .post-link {
  color: red;
}

/* ✅ Специфичность - 0 0 2 1 */
.post > a.post-link {
  color: brown;
}

Селекторы 2-го ранга​

Каждый селектор идентификатора добавляет единицу во второй ранг.

<article class="post">
  <h1 class="post-title" id="title">Lorem ipsum dolor sit amet</h1>
  <p>
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a href="" class="post-link">Читать дальше...</a>
</article>

Цвет текста заголовка будет оранжевым, потому что специфичность второго правила выше.

/* Специфичность - 0 0 1 0 */
.post-title {
  color: green;
}

/* ✅ Специфичность - 0 1 0 0 */
#title {
  color: orange;
}

Селекторы 1-го ранга​

Встроенные стили добавляют единицу в первый ранг, имеют самую высокую специфичность, и переопределить их через CSS невозможно.

<!-- Специфичность такого стиля - 1 0 0 0 -->
<p style="color: green;">
  Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
  veritatis nihil alias iste odit similique sit eius optio veniam, impedit
  cumque fuga facere labore quo id necessitatibus quaerat rerum.
</p>

Одинаковая специфичность​

Если специфичность одинаковая, побеждает правило расположенное ниже в коде.

/* Три селектора элемента с одинаковой специфичностью */
a {
  color: teal;
}

a {
  color: brown;
}

/* Так как это правило стоит ниже, ко всем ссылкам применится оранжевый цвет текста */
a {
  color: orange;
}

Избегаем специфичность​

Если селектор выглядит как A > B > C + X Y, стоит задуматься. Чем проще селектор, тем меньше проблем со специфичностью. Возьмите за правило использовать селекторы класса. Атрибут class не только помогает избежать проблем со специфичностью, но и повышает читабельность HTML-разметки.

Первый подход это присвоение класса общему блоку-родителю и использование селекторов ребёнка и потомка.

<article class="post">
  <h1>Lorem ipsum dolor sit amet</h1>
  <p>
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a href="">Читать дальше...</a>
</article>

Такой CSS хорошо работает когда разметка простая. Но при росте сложности разметки блока, удобная стилизация по селекторам тега становится невозможной. Например, если в посте будет 5 ссылок, а не одна, точечно выбрать какую-то ссылку не получится или селектор будет слишком сложный.

.post {
  /* Стили статьи */
}

.post > h1 {
  /* Стили заголовка */
}

.post > p {
  /* Стили текста */
}

.post > a {
  /* Стили ссылки */
}

.post > a:hover {
  /* Стили ссылки при ховере */
}

Второй подход это добавление описательных классов блоку-родителю и тегам внутри него.

<article class="post">
  <h1 class="title">Lorem ipsum dolor sit amet</h1>
  <p class="text">
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a class="link" href="">Читать дальше...</a>
</article>

Такой CSS неплохо масштабируется и поддерживается. Это отправная точка для написания хорошего CSS. Тем не менее с ростом сложности разметки могут быть небольшие проблемы со специфичностью.

.post {
  /* Стили статьи */
}

.post > .title {
  /* Стили заголовка */
}

.post > .text {
  /* Стили текста */
}

.post > .link {
  /* Стили ссылки */
}

.post > .link:hover {
  /* Стили ссылки при ховере */
}

Третий подход заключается в задании очень специфичных классов блоку-родителю и каждому тегу внутри него по принципу имя_родителя-имя_ребёнка.

<article class="post">
  <h1 class="post-title">Lorem ipsum dolor sit amet</h1>
  <p class="post-text">
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a href="" class="post-link">Читать дальше...</a>
</article>

При этом подходе используются простые селекторы класса, не нужно думать о специфичности. Такой CSS хорошо масштабирутеся и поддерживается. В современной разработке при написании стилей используются вариации этого подхода.

.post {
  /* Стили статьи */
}

.post-title {
  /* Стили заголовка */
}

.post-text {
  /* Стили текста */
}

.post-link {
  /* Стили ссылки */
}

.post-link:hover {
  /* Стили ссылки при ховере */
}
Полезно

Не стоит ограничивать себя в выборе лишь одного подхода. Все три метода именования разметки классами и написания стилей можно использовать вместе, в зависимости от ситуации.

Ключевое слово !important​

Специфичность правила можно повысить при помощи ключевого слова !important, добавив его после значения свойства.

<p class="text" id="text-id">
  Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
  veritatis nihil alias iste odit similique sit eius optio veniam, impedit
  cumque fuga facere labore quo id necessitatibus quaerat rerum.
</p>

Значение такого свойства приоритетнее всех остальных, и цвет текста абзаца будет оранжевый.

p {
  color: orange !important;
}

p#text-id.text {
  color: blue;
}
Внимание

Крайне не рекомендуется использовать !important в современной разработке. Единственный приемлемый случай это переопределение значения свойства, если нет прямого доступа к файлу со стилями, например стиль из библиотеки.

Наследование свойств​

Генетическое наследование привычно видеть в жизни. Например, у высоких родителей чаще всего высокие дети, часто передаётся цвет волос и так далее. Нечто подобное есть и в CSS. Если установить зелёный цвет текста родителю, то, если какое-либо правило не переопределит это значение, цвет текста всех его потомков также будет зелёным.

Наследование это механизм, при помощи которого значения определённых свойств передаются от предка к его потомкам. Наследуются, в основном, свойства оформления текста, например, цвет. Это удобно, так как не нужно задавать повторяющиеся стили для каждого потомка в отдельности.

<article class="post">
  <h1>Популярное за этот месяц</h1>
  <p>
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a href="">Читать дальше...</a>
</article>

Сделаем весь текст поста зелёного цвета.

.post {
  color: green;
}

В инструментах разработчика на вкладке Styles в самом низу увидим наследование для абзаца и заголовка.

В жизни не все атрибуты родителей передаются детям. То же самое и в CSS, не каждое свойство по умолчанию наследуется потомками. Фактически, если бы все свойства были унаследованы, эффект был бы аналогичен отсутствию наследования вообще, и пришлось бы написать очень много CSS для сброса стилей унаследованных от родителя, чтобы переопределить это поведение.

Исключения​

Не все элементы наследуют значение свойств от предков. Например, ссылка не наследует цвет текста.

<article class="post">
  <h1>Популярное за этот месяц</h1>
  <p>
    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
    cumque fuga facere labore quo id necessitatibus quaerat rerum.
  </p>
  <a href="" class="post-link">Читать дальше...</a>
</article>

Если необходимо чтобы в посте она была такого же цвета как и весь текст, можно использовать специальное значение inherit для свойства color, чтобы не дублировать цвет в двух местах.

.post {
  color: green;
}

.post-link {
  color: inherit;
}
Полезно

Значение inherit говорит браузеру о том, что необходимо унаследовать значение свойства от предка. Для цвета предпочтительнее использовать значение currentColor (текущий цвет).

Цвет текста страницы​

Для того, чтобы задать базовый (доминантный) цвет текста всего документа, создаём правило для тега <body> и добавляем свойство color. Этот цвет будет унаследован всем текстовым контентом, кроме ссылок.

body {
  color: #2a2a2a;
}
