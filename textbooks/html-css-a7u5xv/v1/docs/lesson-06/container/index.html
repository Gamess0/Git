Свойства контейнера

Flex-контейнер (дальше просто контейнер) это родитель группы элементов, которым необходимо изменить порядок расположения. Именно свойства контейнера контролируют направление осей, многострочность и позиционирование элементов в строке.

Свойство display​

display: flex | inline-flex

Создаёт flex-контейнер, блочный или строчно-блочный, в зависимости от заданного значения, и устанавливает flex-контекст для всех детей (не потомков) контейнера, превращая их во flex-элементы.

<ul class="menu">
  <li class="item"><a href="" class="link">Главная</a></li>
  <li class="item"><a href="" class="link">Блог</a></li>
  <li class="item"><a href="" class="link">Подкасты</a></li>
  <li class="item"><a href="" class="link">Контакты</a></li>
</ul>

Создадим горизонтальное меню, превратив ul.menu во flex-контейнер. Тогда его дети, элементы li.item, расположатся вдоль главной оси, которая по умолчанию идёт слева направо.

.menu {
  display: flex;
}

Флекс-контейнер ul.menu никак не влияет на a.link внутри элементов списка, потому что они его потомки, а не дети. Флекс-элементами становятся именно дети, в нашем случае элементы li.item.

Свойство flex-direction​

Flexbox это система расположения элементов в одном определённом направлении - горизонтально или вертикально. Свойство flex-direction устанавливает направление главной оси, определяя направление элементов в контейнере.

flex-direction: row | row-reverse | column | column-reverse;
  • row - ось начинается слева и заканчивается справа. Это значение по умолчанию.
  • row-reverse - ось начинается справа и заканчивается слева.
  • column - главная ось становится вертикально, начало сверху и конец снизу.
  • column-reverse - главная ось становится вертикально, начало снизу и конец сверху.
Полезно

То есть это свойство меняет местами main-start и main-end главной оси. Flex-элементы всегда будут размещены вдоль главной оси от начала (main-start) до конца (main-end).

По умолчанию для flex-direction установлено значение row, поэтому элементы верхнего меню начинают выстраиваться слева направо. Для нижнего меню задано значение row-reverse, главная ось развернулась на 180 градусов и элементы выстраиваются справа налево.

Свойство justify-content​

Управляет позиционированием элементов на главной оси, от main-start до main-end.

justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly
  • flex-start - элементы прижимаются к старту оси. Это значение по умолчанию.
  • flex-end - элементы прижимаются к концу оси.
  • center - элементы центрируются на оси.
  • space-between - элементы равномерно распределяются на главной оси. Первый элемент устанавливается в начало оси, а последний в конец.
  • space-around - элементы распределяются равномерно, но левый и правый элемент отстают от границ контейнера на половину зазора между остальными элементами.
  • space-evenly - элементы распределяются так, что расстояние между элементами, и от крайних элементов до границ контейнера одинаковое.
Внимание

У этого свойства есть и другие значения, но они не поддерживаются многими браузерами или работают нестабильно между разными браузерами.

Самые часто используемые значения, в том числе значение по умолчанию, приведены в примере. Они включают в себя сдвиг контента вправо (flex-end), центрирование (center) и равномерное распределение пространства между элементами (space-between).

Свойство align-items​

Управляет расположением элементов вдоль поперечной оси. Это аналог justify-content для главной оси.

align-items: stretch | flex-start | flex-end | center | baseline
  • stretch - элементы растягиваются на всю длину оси. Это значение по умолчанию.
  • flex-start - элементы прижимаются к старту оси.
  • flex-end - элементы прижимаются к концу оси.
  • center - элементы центрирутся на оси.
  • baseline - элементы вырвниваются по базовой линии их текстового содержимого.
Внимание

У этого свойства есть и другие значения, но они не поддерживаются многими браузерами или работают нестабильно между разными браузерами.

Наверное самое частое применение - это вертикальное центрирование элементов в строке, особенно если элементы разной высоты. Добавим к нашей разметке шапку с логотипом и вложим меню в шапку так, чтобы логотип и меню были соседями.

<header class="page-header">
  <a href="" class="logo">Логотип</a>

  <ul class="menu">
    <li class="item"><a href="" class="link">Главная</a></li>
    <li class="item"><a href="" class="link">Блог</a></li>
    <li class="item"><a href="" class="link">Подкасты</a></li>
    <li class="item"><a href="" class="link">Контакты</a></li>
  </ul>
</header>

Первое что необходимо сделать это поставить в строку логотип a.logo и меню ul.menu, для этого превращаем header.page-header во flex-контейнер.

.page-header {
  display: flex;
}

Меню выше, чем логотип (со всеми стилями из примера), поэтому он растягивается по вертикали на всю длину поперечной оси, из-за того что align-items по умолчанию задано значение stretch. Чтобы исправить это и выровнять логотип и меню вертикально по центру - задём значение center.

.page-header {
  display: flex;
  align-items: center;
}

Последним шагом раздвинем логотип и меню по углам хедера, задав justify-content значение space-between. Так как элемента в строке всего два (лого и меню), они прижимаются к краям главной оси, а всё свободное пространство находится между ними.

.page-header {
  display: flex;
  align-items: center;
  justify-content: space-between;
}

Карточка с изображением​

Стандартная проблема - карточка с горизонтально расположенным контентом и изображением. Карточка - flex-контейнер, это значит что align-items по умолчанию задано значение stretch, то есть элементы вертикально растягиваются по высоте самого большого элемента. Если контента больше,, чем высота изображения, оно начинает растягиваться, менять пропорции и искажаться.

Проблема решается просто, достаточно переопределить значение align-items у flex-контейнера. В этом случае подойдёт значение flex-start, чтобы элементы прижимались к началу поперечной оси.

.card {
  display: flex;
  align-items: flex-start;
}

Свойство flex-wrap​

По умолчанию создаётся однострочный контейнер и элементы размещаются на одной строке, сжимаясь по необходимости. Свойство flex-wrap позвоялет превратить однострочный контейнер в многострочный. Элементы будут добавляться в строку до тех пор, пока в ней будет место, после чего будет автоматически создана еще одна строка.

 flex-wrap: nowrap | wrap | wrap-reverse
  • nowrap - все элементы будут на одной строке. Это значение по умолчанию.
  • wrap - разрешает элементам перепрыгивать на другие строки, которые по умолчанию располагаются сверху вниз (вдоль поперечной оси).
  • wrap-reverse - разрешает элементам перепрыгивать на другие строки, которые располагаются снизу вверх (вдоль поперечной оси).

Сетка элементов​

Многострочный контейнер позволяет сделать «сетку» элементов, например карточек продуктов в интернет-магазине, превью статей в блоге, галерею изображений и т. п.

Пошагово разберём технику создания «сеток». Первое, что необходимо сделать - превратить ul.card-set во flex-контейнер и сделать его многострочным. Контейнеру можно задать произвольную ширину.

.card-set {
  display: flex;
  flex-wrap: wrap;
}

Далее используем flex-свойство контейнера gap, которое определяет зазор между столбцами и строками сетки. Пусть в нашем предполагаемом макете это 10px.

.card-set {
  display: flex;
  flex-wrap: wrap;
  gap: 10px;
}
Свойство gap

Свойство gap комбинирует в себе значение свойств row-gap и column-gap, которые позволяют отдельно указать размеры зазоров между строками и столбцами сетки. Используйте gap когда зазоры одинаковые, в противном случае подойдут row-gap и column-gap.

Последний шаг - задать размер flex-элементам li.item. Ширина указывается в процентах от общей ширины родителя (100%) по следующей формуле.

calc((100% - кол-во_зазоров_в_строке * значение_одного_зазора) / кол-во_элементов_в_строке)

Количество зазоров между элементами в строке всегда на единицу меньше чем количество элементов. В нашей сетке три элемента в ряд, значит в одной строке есть два зазора по 10px. Свойство flex-basis это замена width для flex-элементов и рассматривается дальше.

.item {
  flex-basis: calc((100% - 20px) / 3);
}
Почему не использовать фиксированные значения?

Это масштабируемое решение, так как размеры элементов не нужно будет пересчитывать заново при изменении ширины контейнера или значения зазора между элементами.

Свойство align-content​

Управляет выравниванием всех строк многострочного контейнера вдоль поперечной оси, если есть свободное место. Аналогично тому, как justify-content выравнивает элементы в каждой строке. Это свойство не действует, когда есть только одна строка элементов или в контейнере нет свободного места по вертикали (явно не задана высота больше чем контент).

align-content: flex-start | flex-end | center | space-between | space-around | space-evenly | stretch
  • stretch - строки равномерно растягиваются чтобы занять всё место. Значение по умолчанию.
  • flex-start - строки прижимаются к старту оси.
  • flex-end - строки прижимаются к концу оси.
  • center - строки центрируются на оси.
Интересно

Значения space-between, space-around и space-evenly работают аналогично свойству justify-content, только для строк, и используются редко.
